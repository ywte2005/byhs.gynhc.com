## 一、核心配置体系
+ **全配置化设计**：所有等级、奖励、分红、分润均后台可配置
+ **五大配置模块**：
    1. 等级配置 - 自定义等级体系与升级条件
    2. 奖项配置 - 自定义奖励规则
    3. 分红配置 - 自定义分红档位与条件
    4. 分润配置 - 自定义分润规则

## 二、佣金触发场景
### 三大触发场景
1. **进件入驻**：商户缴纳入驻费后触发奖励
2. **刷单任务**：三方推送交易数据后即时触发 分红分润
3. **购买等级权益**：用户购买升级后触发

### 佣金配置特点
+ 灵活设置发放对象（直推、间推、指定层级或全关系树）
+ 多种奖励金额配置（固定金额、比例或混合）
+ 支持等级要求和业绩增量条件作为发放门槛

## 三、互助任务机制
### 核心流程
1. 商户发起互助任务（设置目标金额、时间范围）
2. 缴纳保证金，任务拆分为多个小额任务
3. 系统根据互助余额分批分配任务
4. 实时监控余额，保证金低于阈值暂停分配
5. 任务完成，释放保证金 服务费从保证金里面扣除 有相关明细

### 互助余额机制
+ **核心原则**：先刷别人，才能被别人刷
+ **保证金余额公式**：当前余额 = 初始余额 + 刷别人总额 - 被别人刷总额-减掉自己的手续费+刷他人产生的佣金
+ **风控作用**：防止过度透支， 

## 四、交易分润体系
+ **核心规则**：按等级差分润，等级差越大，分润比例越高
+ **多重分润档位**：可配置多级分润规则
+ **分润条件**：等级差要求、业绩增量条件可组合配置
+ **计算流程**：交易发生→计算等级差→匹配分润档位→检查条件→发放分润

## 五、业绩与增量统计
+ **业绩三重作用**：
    1. 触发即时奖励
    2. 累计为个人/团队业绩（用于分红）
    3. 计算月度增量（作为奖励条件）
+ **增量计算**：月度业绩增量 = 本月总业绩 - 上月总业绩
+ **增量应用**：所有奖励和分润均可将增量作为配置条件

## 六、分红体系
+ **多档位分红**：可配置多个分红档位，用户可同时达标多个档位
+ **达标条件**：团队业绩门槛、个人达标额度、达标人数、业绩增量
+ **奖池分配**：奖池 = 刷单总金额 × 奖池比例，达标用户平分奖池
+ **发放周期**：每月1号统计，3号前完成发放  后台自定义时间

## 七、商户等级体系
+ **灵活等级配置**：名称、数量、排序完全自定义
+ **双重升级方式**：
    1. 购买升级：支付升级费立即升级
    2. 业绩升级：达到配置的业绩要求自动升级
+ **多维升级条件**：可配置个人业绩、团队业绩、直推人数等条件



推广体系
1. 平台采用多层级推广关系，推广层级、发放对象与规则均支持后台配置（可新增/修改/启用/停用）。
2. 推广权益与收益与用户等级绑定：用户需购买/升级到对应等级；等级名称、数量、排序、升级方式与条件均可配置。
3. 各层级可分别配置升级条件与收益结构：直推、间推、团队分红/分润等，并可叠加等级门槛、月度业绩增量门槛。
4. 邀请/绑定支持二维码邀请与小程序转发。

奖励类型
1. 直推奖：关系树第1层触发发放。
2. 间推奖：关系树第2层触发发放；可配置发放条件（如等级门槛、增量门槛等）。
3. 级差奖（等级差分润）：订单/交易产生后，若上级等级高于下级，则按等级差匹配分润档位，获得对应差额/比例收益。
4. 平级奖：面向同等级（省级）设置平级奖励规则，同等级查找仅发放第一个平级。
5. 团队业绩分红：按月基于个人/团队业绩与增量达标情况进行分红，档位与奖池比例可配置。

分佣来源
1. 代理等级付费/升级费（购买升级可触发升级奖励）。
2. 互助任务手续费（交易数据推送后结算，并计入个人/团队业绩用于分红统计）。
3. 第三方支付分润（按等级差规则配置档位与比例后发放）。
